# Starship Quest -- ADRIFT 5 walkthrough regression script.
#
# Source: the game's OWN in-game WALKTHROUGH command (task 'Walkthroui',
# command [wlkthrgh], Larry Horsfield), extracted verbatim and split one command
# per line, then ADAPTED to route around the stale 2019 snapshot's drift (the
# game was revised after the built-in walkthrough was written, so it desyncs --
# it also fails identically in FrankenDrift).  Port-accurate ship dirs
# (po/st/fo/af), unlike the old CASA original-release solution (26 unparsed cmds,
# ended at turn ~53).
#
# Start-options preamble: O to continue to the Start Options page, B to begin
# from the first location.  The walkthrough's first command `n` answers the
# "Do You Wish To Explore The Ship First? (Y/N)" prompt with No.
#
# ADAPTATIONS (route around stale drift; all verified vs FrankenDrift):
#  - Both mid-game shuttle "test" launches: `remove backpack`+`drop backpack`
#    before `sit in chair`, `get backpack`+`wear backpack` after `stand up`
#    (the "can't sit wearing/carrying the backpack" restriction post-dates the
#    built-in walkthrough).
#  - Bearion chase rebuilt: you must first SEE the bearion (so `SeeBearion`
#    completes), THEN tame the Rhinopine at End Of Path with nuts and
#    `lay trail of nuts` to lure it to Bend By Tree, so climbing the tree lures
#    the bearion into the clearing (`ReachTree`, not the death `ReachTreeY`);
#    then `dodge bearion` at In Wide Clearing.
#  - Rhinopine feed grafted back in for the LAST 5 POINTS: `lay trail of nuts`
#    does not consume the brown nuts -- it moves them onto the tree branch at
#    the bend (`MoveObject Nuts1 ToLocation SittingOnB`).  So after the lure:
#    climb into tree / get nuts (re-pick; the repeatable PickNutsSc +5 does NOT
#    re-award -- confirmed identical in FrankenDrift) / put nuts in pocket
#    (hands must be free to climb) / d / get nuts (GetNutsFro: the rhinopine
#    sniffs them) / feed rhinopine (DropNutsRp1 +5).  The RP stays under the
#    tree for the collision rescue, so the feed and the lure are NOT mutually
#    exclusive after all.
#  - Stone-hut sacrifice: after `climb on beam`, ONE `z` (wait for the native to
#    enter and pick up the dropped amulet), THEN `d` (drop onto him) -- the
#    built-in "climb on beam - d" climbed straight back down.
#  - Tigerilla skin: `get skin` (the built-in typed `get skin and tail` -- the
#    word "tail" is unknown in this build, so the whole command errored and the
#    skin was never picked up, leaving `wrap vegetation in skin` with the skin
#    out of reach -> "will not help you in this game").  The wrap now completes.
#  - Endgame shuttle launch: `get gold disc` (out of the backpack) BEFORE
#    `remove backpack`/`drop backpack` -- the built-in dropped the backpack (with
#    the discs inside) first, so `put gold disc in slot` failed ("not carrying")
#    and the launch buzzed "INSERT COURSE TAPE!".  With the disc in hand the whole
#    Galaxis docking / matrix / dome / engage-drive endgame plays through.
#
# RESULT: deathless *** CONGRATULATIONS! *** -- "scoring the maximum 800
# points!" in 872 turns.  FrankenDrift replays it byte-identically in BOTH RNG
# modes (MATCH 0|0).
#
# Run with:
#   test/a5_groundtruth.sh test/adrift5-games/StarshipQuest.blorb test/StarshipQuest_walkthrough.txt
