# The Spectre of Castle Coris -- the game's OWN built-in walkthrough (WLKTHRGH).
#
# Source: extracted verbatim from the WLKTHRGH command's completion text (task
# key "Walkthroug"), one command per line, with the inline dashed annotations
# ("- the goat tries to butt you -", "-chased by dog -") stripped and the
# intro "o"/"b" handshake prepended.  This is the AUTHOR's full solution and
# ends "YOU HAVE WON!" in his own text.
#
# STATUS: WORK IN PROGRESS -- this does NOT yet win under a faithful engine.
# The built-in walkthrough is itself buggy: its navigation was written against
# an EARLIER map revision and desyncs in this build (it even contains a literal
# stray-comma typo "w, ,sw" in the castle-grounds leg).  FrankenDrift (the
# reference ADRIFT 5 runner) fails it at the IDENTICAL point Scarier does, so
# the fault is the walkthrough text, not the engine.
#
# Fixes already applied here vs the raw WLKTHRGH text:
#   * Castle-grounds route corrected: from "Inside Main Gates" the only exit is
#     NW, so the garbled "w, ,sw, w, w, e, s" -> "nw, sw, w, s" (to By Woodpile,
#     via By Fountain -> By Cottage Garden -> In Cottage Garden).  This alone
#     advanced the run from turn 160 to the castle interior.
#
# Two mechanics a completing script must honour (learned the hard way):
#   1. THE SPECTRE is a repeating turn-based killer (region-varying period,
#      10-35 turns; warning at +26, death at +30).  It is banished ONLY while
#      the prayer book is HELD IN HAND (book-in-bag fails: "You are not carrying
#      the prayer book!").  The un-ambiguous banish command is "read prayer"
#      (ReadObjects override "ReadTheLor" on the Prayer object) -- plain
#      "say prayer" is captured by "say <x> to <NPC>" when a character is present
#      (e.g. deaf Cissy Green), so it must NOT be relied on near NPCs.
#   2. Every two-handed action (get wood, climb rope, dig, ...) forces the book
#      DOWN for a turn; if the Spectre's timer fires in that window it is lethal,
#      so the book must be re-grabbed and a prayer read the instant it appears.
#      Because navigation fixes shift turn counts, the whole prayer cadence has
#      to be re-derived -- an adaptive re-run solver (see
#      TODO_a5_walkthrough_bugs.md, "Spectre of Castle Coris") does this.
#
# REMAINING BLOCKER: pervasive map-navigation desync in the castle grounds and
# interior (the gatehouse windlass/portcullis puzzle desyncs, so "n" under the
# gatehouse is refused -- "you cannot enter ... until you have raised the
# portcullis").  Completing the win needs a full region-by-region re-derivation
# of the navigation against this build's map (see the map dump in the TODO).
