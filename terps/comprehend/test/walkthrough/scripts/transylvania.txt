# Transylvania -- full winning walkthrough (works on BOTH releases).
#
# One script, two games:
#   * gameid "transylvania"   -- Penguin Software's original (1985 Apple II woz)
#   * gameid "transylvaniav2" -- Polarware's IBM/PC re-release (1987, the
#                                TransylvaniaPC DOS files)
# They share their map, story and parser, so the same inputs win both. This
# script is run against each in run_walkthroughs.sh.
#
# Two notes that matter on the 1985 Apple game specifically (both harmless on the
# DOS re-release, which is why the one file covers both):
#
#   * DROP CROSS once the vampire is destroyed. The 1985 inventory limit is one
#     slot tighter, so the spent cross has to go or GET RING (in the vault) is
#     refused with "You are carrying too much." (On v2 the cross can be kept;
#     dropping it is just tidy.)
#   * The Apple v1 game depends on the _strings2 alignment fix in
#     GameData::load_extra_string_files (two leading placeholders for gameid
#     "transylvania"). Without it the 1985 disk's MA-ME string files sit two
#     slots off, so "shoot werewolf", "sing" and "show cross" not only printed
#     the wrong lines (the broom flight, "the window is shut", "a bottle of
#     clear liquid") but their kill/idle effects desynced -- the werewolf and
#     vampire could never be killed and the ring stayed barred (the game was
#     unwinnable). With the shim the deterministic run below reaches "Well Done!!!".
#
# Puzzle notes (the dark-cellar bar, the deterministic werewolf coffin-window,
# the wizard's duet for the cross, the vampire at the entranceway, the mice that
# ground the eagle, the flying saucer at dawn) are inline below.
#
# The whole run is deterministic: in determinism (testing) mode the engine seeds
# the shared erkyrath_random RNG with a fixed seed (1234), so the rand-gated
# werewolf/vampire dice fall the same way every run. Verified byte-identical
# across runs, on each game. (The headless harness forces determinism whenever a
# COMPREHEND_SCRIPT transcript is being replayed.)
#
# Win marker (see run_walkthroughs.sh): "Well Done!!!"
# One input per line; blank lines and #-comments are ignored.
