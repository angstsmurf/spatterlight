# The Crimson Crown -- full winning walkthrough (Penguin/Polarware, 1985 Apple II).
#
# Drives the two-disk game from the lake shore all the way to the win -- the
# merchant-ship rescue and the closing "THE END." epilogue, exactly where the
# real Apple II game stops (verified on the woz in MAME). The party (you, Erik
# and the sorceress Sabrina) recovers the regalia, frees the dragon Fury, and
# destroys the vampire to restore the crown. The interpreter switches from side A
# to side B by itself when disk one is finished (selectDiskByGameMagic), so only
# the side A .woz needs to be named on the command line.
#
# The whole run is deterministic: the engine never seeds rand(), so it rides the
# default srand(1) stream and is reproducible across runs (verified identical).
# That matters here because the endgame is a timed escape with a per-turn random
# element (Erik shaking off the vampire's spell), so the exact move sequence is
# what lands the win -- see the endgame notes at the bottom.
#
# Two engine bugs on the disk-two endgame path were fixed to reach this win (both
# only reachable once a walkthrough drives the game this far):
#   * setupDisk() left _gameStrings null on disk two, so any disk-two death (e.g.
#     losing the vampire's battle of wills) crashed handle_restart() on a null
#     deref. Disk two shares the restart-prompt index, so it now stays valid.
#   * The win path printed strings 0x21c/0x21d, two off from the real victory text
#     -- it showed the troll/mud daemon lines instead of the merchant-ship rescue
#     (0x21a) and the closing epilogue that ends "THE END." (0x21b). Corrected.
#     (The original game stops on "THE END." -- confirmed on the woz in MAME; the
#     adjacent "...completion code is @." string is a dead contest stub the real
#     game never prints, with the '@' replacement marker left unset/blank.)
#
# Win marker (see run_walkthroughs.sh): "THE END."
# One input per line; blank lines and #-comments are ignored.
